# 4.HTML转flutter的建议

该 HTML本身有很多缺陷——没有良好的响应式布局、没有真正的功能联调，这意味着参考代码来还原flutter效果会很糟糕，会错上加错


HTML 原型本身“不完善”，常见问题是：

* 像素/绝对定位硬编码、响应式缺失
* 样式与布局混杂在 DOM 结构里（为了快而写）
* JS 用 DOM 操作拼 UI、拼 SVG、拼 class 切换

  这些做法在 Flutter 里照搬会把“原型缺陷”带过去，而且会让组件分层变差。


**Flutter 复刻的唯一真相来源是：截图（full + component）与页面/组件功能手册。**

HTML/CSS/JS 只作为“结构线索”和“信息补完”， **不作为视觉实现的直接依据** 。

**跳转关系可以“全盘参考” HTML** ，因为这是信息结构，不是视觉实现。

文案、字段名、mock 数据形状（保留为“内容与结构参考”），可以保留
